- 本次验收日期：2026-04-02
- 本次验收结论：**通过**
- 验收入口统一使用主域名：`http://ad_tools.thinkpro.top`

## 2. 验收项 A：统一上传入口（API 闭环）

### A1. 不带文件：验证“路由存在且到达网关”

**命令**

```bash
curl -i -X POST "http://ad_tools.thinkpro.top/api/upload-image"
```

**预期**

- HTTP 400
- JSON 提示缺少 `file` 字段

**实际留痕**

- `HTTP/1.1 400 Bad Request`
- `{"success":false,"error":"没有上传文件，请使用 form-data 并传入字段 file"}`

### A2. 带文件：验证“真实上传到 OSS 成功”

**命令**

```bash
curl -i -X POST "http://ad_tools.thinkpro.top/api/upload-image" \
  -F "file=@/root/work/tuchuang-backend/test-assets/demo.svg;type=image/svg+xml"
```

**预期**

- HTTP 200
- JSON：`success:true` 且包含 `url/key/originalName/mimeType/size`

**实际留痕（示例）**

- `HTTP/1.1 200 OK`
- `url=https://my-project-hangzhou-imageup.oss-cn-hangzhou.aliyuncs.com/uploads/2026/04/02/1775103147000-demo.svg`
- `key=uploads/2026/04/02/1775103147000-demo.svg`

### A3. 旧接口下线验证（防串线）

**命令**

```bash
curl -i -X POST "http://ad_tools.thinkpro.top/api/upload-to-imgfi"
```

**预期**

- HTTP 404
- `Cannot POST /api/upload-to-imgfi`

**实际留痕**

- `HTTP/1.1 404 Not Found`
- `Cannot POST /api/upload-to-imgfi`

## 3. 验收项 B：OSS 外链可访问（B1）

> 目的：证明上传返回的 `url` 在公网可读，且不是“200 但内容错误页”。

### B1. HEAD 验证（推荐）

**命令（用 A2 返回的 url 替换）**

```bash
curl -I "https://my-project-hangzhou-imageup.oss-cn-hangzhou.aliyuncs.com/uploads/2026/04/02/1775103147000-demo.svg"
```

**预期**

- HTTP 200
- `Content-Type` 与文件类型一致（例如 svg 为 `image/svg+xml`）

**实际留痕**

- `HTTP/1.1 200 OK`
- `Content-Type: image/svg+xml`
- `Content-Length: 308`
- 备注：若出现 `Content-Disposition: attachment` / `x-oss-force-download: true` 属正常现象（强制下载不影响外链可用性）。

### B1. 内容片段验证（防“200 但其实是 HTML 错误页”）

**命令**

```bash
curl -s "https://my-project-hangzhou-imageup.oss-cn-hangzhou.aliyuncs.com/uploads/2026/04/02/1775103147000-demo.svg" | head
```

**预期**

- 能看到 `<svg ...` 开头内容

## 4. 验收项 B2：/temp 回显路径可访问

> 目的：解决“页面导入的图片不显示/拼接结果不显示”这类问题，核心验证点是 `/temp/`、`/uploads/` 是否被主域名入口正确代理到融合平台。

**页面操作**

- 打开：`http://ad_tools.thinkpro.top/ronghe/image-stitch`
- 导入素材 → 拖拽到槽位 → 生成拼接图片
- 记录页面给出的生成路径，通常类似：
  - `/temp/<sid>/stitched_<timestamp>.jpg`

**验证方式**

- 将上面的相对路径拼到主域名访问：
  - `http://ad_tools.thinkpro.top/temp/<sid>/stitched_<timestamp>.jpg`

**预期**

- 浏览器能直接打开/下载图片，不应跳转到 FB 首页或返回 404。

## 5. 验收项 C：页面端到端体验

**页面操作**

- 在 `ImageStitch` 页面完成：
  - 导入素材（缩略图可见）
  - 生成拼接（预览可见）
  - 上传到外链（返回外链 URL）
  - 打开外链 URL（OSS 可访问）

**预期**

- 控制台不应出现 `Cannot POST /api/upload-image` 这类 404
- 上传失败时应返回网关 JSON 错误信息（而不是 HTML 错误页），方便定位

## 6. 常见失败形态与定位

- **400 JSON（缺 file）**：正常（说明路由可达）。
- **404 Cannot POST /api/upload-image**：路径没到 `api-gateway`（入口层分流或网关路由缺失）。
- **502 Bad Gateway**：Nginx 上游不可达（常见于容器重建后 upstream 缓存旧 IP；重启 `ronghe-nginx` 可恢复）。
- **页面不显示 /temp 图**：主域名入口没有把 `/temp/`、`/uploads/` 代理到融合平台。
